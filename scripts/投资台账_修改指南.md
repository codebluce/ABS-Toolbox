# 投资台账模块 · 修改指南（给维护 / 改版的 Agent）

本模块共 3 个源文件，都在 `skills/ABS工具箱/scripts/`。改完**重跑 `gen_integrated_dashboard.py`** 即生效。
**不要**改已生成的 `ABS综合看板_*.html`（产物，会被覆盖），也不用管设计项目里的 `投资台账.dc.html`（那是 React 原型，与线上不同源）。

## 文件职责（先认清改哪一个）

| 文件 | 管什么 | 语言 |
|---|---|---|
| `gen_investment_ledger.py` | **数据**：从台账取哪些列、口径、字段名、每条记录的键 | Python |
| `itl_panel.js` | **逻辑 + 结构**：筛选规则、分组/透视、汇总卡、三张表、导出、面板 HTML、文案 | vanilla JS |
| `itl_panel.css` | **视觉**：颜色、间距、字号、表格/卡片样式（全 `.itl-` 前缀） | CSS |

数据流：`compute_data`(py) 产出 records → `render_body`(py) 把 records 塞进 `window.ITL_DATA` → `itl_panel.js` 读它渲染。
**关键原则**：records 的键（如 `asset`/`inst`/`share`）是 py 与 js 的合同，两头必须用同一个键名。

## 每条记录有哪些字段（compute_data 产出）

```
asset 资产类型D · proj 项目名称E · venue 发行场所F · mgr 计划管理人G
underwriter 联席承销商H · custodian 托管行I · scale 规模J · tenor 期限K
date 簿记时间L(YYYY-MM-DD) · layer 分层情况P · rating 评级S
inst 认购机构(穿透优先) · cost 成本(小数,0.0195=1.95%) · share 认购份额V(亿)
cd 国股CD · spread 利差=cost-cd
```
想用一个**当前没带**的列（如「认购倍数O」），先在 `gen_investment_ledger.py` 的 `compute_data` 里给 records 加这个键，再到 js 里引用。

## 常见改动配方

### A. 加/减一个「快速多选」筛选字段（值有限，如资产类型/分层/评级）
**文件：`itl_panel.js`** → 顶部 `CHIP` 数组。
```js
var CHIP = [
  { key: 'asset', label: '资产类型', col: 'D' },
  { key: 'venue', label: '发行场所', col: 'F' },   // ← 新增一行即可
  ...
];
```
前提：records 里已有 `venue` 键（本例已有）。芯片、计数、筛选、清空全自动接上，无需改别处。

### B. 加/减一个「关键词联想」筛选字段（值多，如管理人/机构）
**文件：`itl_panel.js`** → 顶部 `KW` 数组，加一行 `{ key, label, col }`。同样要求 records 有该键。输入框、联想下拉、多选芯片自动生成。

### C. 加/减一个「区间」筛选（数值或日期）
**文件：`itl_panel.js`**，三处都要动（照现有 `share`/`date` 抄）：
1. `build()` 里 `itl-ranges` 区块加一组 `<input>`（给唯一 id）。
2. `build()` 底部给该 input 加 `addEventListener('input', ...)`，写进 `st.你的键`。
3. `filtered()` 里加一段区间判断。
4. `st` 默认值 + `reset()` 里清空。

### D. 加/减一个「分组维度」或「透视行列维度」
**文件：`itl_panel.js`** → 顶部 `DIMS` 数组加一行 `{ key, label }`。分组统计、透视表的行/列按钮都从这个数组来，一处加、三处生效。

### E. 改「汇总卡」（那 5 张：记录数/份额/项目数/成本/利差）
**文件：`itl_panel.js`** → `renderCards()` 里的 `cards` 数组。改文案、加卡、换算法都在这。（改卡片间距/宽度去 css 的 `.itl-cards` / `.itl-metric`。）

### F. 改「分组统计表」的列或排序
**文件：`itl_panel.js`** → `renderGroup()`（表头 + 行）与 `groupData()`（算指标）。新增可排序列：表头加 `<th class="s" data-act="sort" data-key="你的键">`，`groupData()` 产出该键即可。

### G. 改「透视表」（默认认购机构×资产类型、取前 14 行×9 列）
**文件：`itl_panel.js`** → `pivotData()`（`slice(0,14)` / `slice(0,9)` 改数量，聚合值默认 `share` 合计，可换 `count` 等）与 `renderPivot()`（呈现）。

### H. 改「明细清单」的列
**文件：`itl_panel.js`** → `renderDetail()`（表头 `<th>` + 行 `<td>`，用 records 的键）。显示上限在 `slice(0, 500)`。

### I. 改「导出 CSV」的内容
**文件：`itl_panel.js`** → `exportCSV()`，三个分支对应 分组/透视/明细 三视图，各自的表头与行独立控制（导出为全量，不受表内 200/500 显示上限限制）。

### J. 改默认视图 / 默认分组维度 / 默认排序
**文件：`itl_panel.js`** → `st` 初始对象：`view`('group'|'pivot'|'detail')、`groupDim`、`sortKey`/`sortDir`、`pivotRow`/`pivotCol`。

### K. 改标题、副标题、页脚、区块小标题等文案
**文件：`itl_panel.js`** → `build()` 里的 HTML 字符串（`itl-banner-title` / `itl-banner-sub` / `itl-grouplabel` / `itl-footer`）。

### L. 改视觉（配色/间距/字号/表格线/圆角）
**文件：`itl_panel.css`**。主色改 `#1a3a5c` 的地方（`.itl-chip.on`、`.itl-badge`、`.itl-bar-fill`、`.itl-accent` 等）。**全部类名带 `.itl-` 前缀**，不会污染看板其它面板；同理别去掉前缀，否则可能与别的 panel 撞样式。

### M. 改数据口径 / 取数逻辑（穿透规则、利差算法、剔除规则、日期纠错）
**文件：`gen_investment_ledger.py`** → `compute_data`。取数走 `abs_common.load_and_filter`（穿透优先、剔自持/黑名单、机构名归并、2025→2026 纠错都在共享底座里）。改口径优先考虑是否该改 `abs_common`（会同时影响其它面板，慎重）。

## 改完必做的自检

1. 重跑：
   ```bash
   PYTHONUTF8=1 python skills/ABS工具箱/scripts/gen_integrated_dashboard.py \
     "skills/ABS工具箱/deliverables/ledger/03_final/2026年ABS发行台账-0703-定稿.xlsx"
   ```
2. 控制台应见 `[QC] … 11 个 panel …`（若你增删了别的 panel，记得同步 `gen_integrated_dashboard.py` 里 `panel_count == 11` 的期望值）。
3. 打开产出 HTML → 「投资台账」Tab：筛选、三视图切换、导出 CSV 均正常，无 JS 报错（看浏览器控制台）。

## 最容易踩的坑
- **键名不对齐**：js 里引用了 `compute_data` 没产出的键 → 该列全空。加字段务必两头同名。
- **改错文件**：想调样式却去改 js 里的内联 style（有少量），或想调逻辑却翻 css——按上表定位。
- **动了共享底座**：改 `abs_common` 会波及发行定价/机构统计等所有面板，不是本模块私事。
- **改产物**：直接编辑 `ABS综合看板_*.html` 无意义，重跑即覆盖。
